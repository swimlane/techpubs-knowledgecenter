Create the Initial Workflow

Workflow automations are connected to applications. The workflow automation tree is evaluated from left to right every time the record is modified (or initially first viewed).

There are three components that make up Swimlane workflow:

  • Stages: Conditions that must be met to trigger or initiate an action. (These are your "if" statements.)
  • Actions: Automated tasks (These are your "then" statements/actions.)
  • Repeats: Repeats iterate over a Text List or Numeric List field and perform an action for each value in a list. (The Quick Start scenarios do not use Repeats in workflow.)

To create workflow:

  1. From your newly-built application, select Workflow from the Application Builder toolbar.

    The workflow page opens. You should see the start of the workflow, which is your application, and an initial, default condition.

  2. Select the workflow condition titled Default Action(s), then click the icon for Create a new action.

  1. Edit the Action properties as shown here:

    When done, click Save.

    This default action will populate the Description text field when a record is first viewed.

  2. To see the workflow in action, from the global navigation menu, hover over Application Records. Select + to add a new record.

  3. A new record is added. Click Save.

    Note: The description field contains the text that you set up in workflow.
