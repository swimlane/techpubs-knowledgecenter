Upload and Manage Plugins

Plugins enable you to perform out-of-the-box integrations with third-party vendors.

To view installed plugins, from the Integrations page, click the Plugins tab.

Upload Plugins

To upload a new plugin:

  1. From the global navigation menu in Swimlane, select Integrations.

  1. From the Integrations taskbar, click the Plugins tab and then click the plus menu icon.

  2. In the Upload Plugins dialog, drag and drop the plugin files or click Browse to locate and select the plugin files.

    3. On the Upload Plugins dialog, a list shows the uploaded plugins and the install status.

    Now your plugins are uploaded to Swimlane.

    When you upload a plugin, you introduce the library or utilities for interacting with the third-party software into Swimlane. For most plugins, your next step is to create an asset with credentials from the plugin library.

Identify the Version of Installed Plugins

You can identify the plugin by either of the following two methods.

Locate the version of the plugin in the list of installed Plugins.

Locate the version of the plugin in the list of Tasks.

Click on the related Task and see the Plugin version under Task Type.

Upgrade Plugins

When uploading a newer version of a plugin, if a breaking issue is detected the status on the Upload Plugin dialog indicates that a new plugin version has been detected. This allows you to upgrade the plugin to the new version without losing previously defined tasks, assets, and configurations. If no breaking change is created by the upgrade, the plugin automatically upgrades without any additional input.

To upgrade a plugin:

  1. Click Upgrade Now to the right of the plugin.

    2. The Upload Plugins dialog opens to show the tasks and assets that upgrading the plugin will affect.

  1. Click Continue Upgrade.

  1. A dialog opens stating "You are about to upgrade a plugin which will create breaking issues." Click Ok.

    2. The Upgrade Plugin window opens showing the tasks and assets affected by the plugin upgrade. This allows you to view and resolve issues.

  1. Click Resolve # Issues to the right of the item.

  1. To resolve issues associated with a task/asset, navigate to the broken parameter.

    2. Note: There can be more than one task/asset to resolve.

    Note: This example uses a broken output parameter. So, we navigate to the OUTPUT PARAMETERS tab.

  1. Click Discover Parameters.

  1. On Discover Parameters, click Run Task to display the discovered parameters' input values.

  1. Select the parameters that the application alerted, and then click Add Selected Parameters.

    2. Tip: A red alert icon indicates unmapped parameters.

    Important! This adds the parameter, but we still need to map it!

  1. Click Unmapped Parameters to verify the unmapped parameter.

  1. Click Add Mappings to App.

  2. Click Approve to add the new fields to the application.

    The parameter is updated, which shows as mapped.

  3. Click Save.

  4. Close out of the remaining screens, navigate back to APPLICATION RECORDS, and select the desired application and add a new record. The new field is available.

  5. To test the record, click Run Search.

    The search runs successfully!

Delete Plugins

To delete a plugin:

  1. On the plugin, click the ellipsis menu and select Delete Plugin.

    2. The Delete Plugin dialog opens to show the tasks and assets deleted and the applications, applets, and forked tasks affected by removing the plugin.

  1. Click Delete Plugin.

  2. A dialog opens stating "You are about to delete a plugin. This will create breaking issues." Select Delete Plugin.

    3. Note: Once the plugin is deleted, the breaking issues noted by this dialog need to be fixed.

    Now the plugin has been removed.
