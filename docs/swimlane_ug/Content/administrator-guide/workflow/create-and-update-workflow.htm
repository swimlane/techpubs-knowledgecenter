Create and Update Workflow

Before you begin creating workflow, you should have a clear idea of the process you want to design. This includes understanding the many conditions and actions you have available to you.

As you build your workflow, you'll create branches of conditions, repeats, and actions.

Icon Definition
Creates a new condition
Creates a new action
Creates a new repeat action
Disables the branch, condition, or action
Deletes the branch, condition, or action

To create or update workflow:

  1. Open an applet or application, then select the Workflow icon from the toolbar.

  1. The application's workflow opens with a default start icon, which can be followed by multiple conditions, including the default condition, Default Action.

    2. The default action only evaluates to true the very first time that a workflow is carried out on a new record. For records created by a client script, the default action is carried out just after the record is first submitted.

  1. To add a new condition, select the Start icon (the root of the workflow) and then click the icon for Create a new condition.

    2. Next you will begin to define the decision points of the workflow from the condition you just created.

  1. On Condition, provide a name and then indicate the field(s) from the applet or application that are part of the process you are designing.

  1. Click Add additional condition to add more than one condition to the flow.

    2. Once you add an additional condition, specify how the conditions relate to each other by setting the Evaluation Type. Set the field to AND to indicate that all the conditions defined within should be met in order to execute the branch. Set it to OR to specify that a branch should be executed upon one of the conditions being met.

    Note: You cannot add multiple conditions to Repeats in workflow.

  1. Click Save to save the condition.

    2. Next, you will add actions to your condition.

  1. Select the condition you just created and then click the icon to Create a new action.

  2. On Action, name the action, then select the action type.

    3. The editable fields vary depending upon the action type that you select. For detail on setting up the various actions, see Select Action Type..

  1. Click Save.

  2. If your action is a Trigger Integration, Trigger Notification, Export and Email Record, or Set Field Value type, then you also have the ability to set up Usage Metrics for the action. Under Integration ROI Calculation Metrics fill out the Manual Time Expenditure and Personnel Costs fields so that the value of the automation is calculated in Usage Metrics.

    3. You must save the workflow before you can access the Usage Metrics section of the App Settings.

  1. Click Apply Metrics, and then click Save.

Setting Up Repeat Actions

Repeat actions are an optional part of workflow setup, and they work exclusively with Text List and Numeric List fields. Use a repeat action when you want a specific integration task action to run over a list of values within a Text List field or a Numeric List field. The Repeat runs the integration task action once for each item in the List field that meets the criteria that you specify when setting up workflow.

Repeats must be preceded by a Condition and must be followed by an action of a Trigger Integration action type.

To set up repeat action:

  1. Add or select a condition in your application's workflow.
  2. In the workflow properties, assign the condition a name and a condition.
  3. Select the condition you just created and then click the click the icon to Create a new repeat.

  1. Next, designate the field you want the integration action to iterate over.

    Note: Repeats only reference Text List, Numeric List fields and multiple-select or grid reference fields.

  2. Select the repeat you just created and then click the icon to Create a new action.

    3. The platform automatically creates a new Trigger Integration action. Set it up so it references the list (or reference) field you designated in the Repeat.

    Note: The integration task must be set up with the input and output mappings needed to complete the purpose of your workflow branch.

  1. If you'd like to be notified when the integration runs, select Send Success Notification or Send Failure Notification, and then compose the notification message.

    2. For more information on sending notifications, see Send Notification Messages.
