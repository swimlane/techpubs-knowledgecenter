Send Notification Messages When Integrations Run

When you create an integration workflow action item, you can also set the action to notify you with an email upon the success or failure of the triggered action.

To send notification messages:

  1. Select a trigger action item in your workflow.

  2. On Action, toggle on the Send Success Notification and/or Send Failure Notification switch.

  1. Click Compose Success Message and/or Compose Failure Message.

  2. Click Recipients and select the user or group you want to be notified.

    3. If you have user or group fields in your applications, they will be the default values in the Recipients field. If you want to select an individual user or group, type the name in the Recipients field and then select the individual user or group.

  1. Enter content into the Subject and Body fields and click Apply.

    2. You can use Mustache templating language to create more complex outputs.

    With your cursor in either the Subject or Body field, click Include Parameters to select parameters from the record or the system that will be automatically added to the notification. These parameters help you identify critical information about running the integration, for example, when the integration ran or ended, or who kicked off the integration.

    The parameters you can add are:

    Parameter References
    Task The task related to the integration
    Action The workflow action related to the integration
    Started The time the integration began
    Finished The time the integration ended
    User The user who ran the workflow, whether by modifying a record or triggering the integration
    Error A system-generated error message
