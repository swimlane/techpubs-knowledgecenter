Pod Requests and Limits

The following are the recommended requests and limits for each pod type for the deployment sizes defined in the Cluster Resources section of System Requirements for an Existing Cluster Install. Requests and limits can be raised based on your cluster's scheduling needs and resource utilization. See Managing Resources for Containers in the Kubernetes documentation for more information.

Note: The MongoDB init containers stop running after their initialization is complete and the primary MongoDB container starts. Their resources are only used during pod start up.
