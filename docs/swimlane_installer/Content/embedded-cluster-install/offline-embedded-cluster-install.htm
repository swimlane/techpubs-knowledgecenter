Offline Embedded Cluster Install

If you need to do an offline installation of Swimlane with the Swimlane Platform Installer, you will need the offline installer package, and an airgap bundle. The airgap bundle works with the Installer to allow an offline install.

Airgap installs require a jumpbox that has access to:

  • the internet
  • the airgapped network
  • a browser, from which you can access the Swimlane Platform Installer

Before you begin, copy the offline installer package to the jumpbox and then to each node in the cluster.

To perform an offline install with the Swimlane Platform Installer:

  1. Copy the offline installer package to the instance that will become the first cluster primary node and untar the package with the following command:

    2. tar zxvf <your file name>.tar.gz

  1. Next, run this install command:

    2. cat install.sh | sudo bash -s airgap ha

  1. Once install completes, save the URL and password that the install generates. In addition, copy the add primary node command to use to add additional masters.

  2. Copy the offline installer package to an additional primary node.

  3. Untar the offline installer package.

  4. Run the add primary node command that you copied in step 3, above.

  5. Repeat steps 4 through 6 above until all nodes have been installed.

  6. Next, run kubectl get nodes to make sure each node is ready.

  7. Copy the airgap bundle to the jumpbox.

  8. In a Chrome browser, enter the URL and password for the Swimlane Platform Installer UI (from the original install command). Upload your .yaml license file and proceed until you are prompted to upload the airgap bundle.

  9. Upload the airgap bundle.

    10. After the airgap bundle is uploaded and processed, see the Configure the Swimlane Platform for an Embedded Cluster Install topic to configure and deploy Swimlane.
